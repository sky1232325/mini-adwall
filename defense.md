## 2. 项目概览
- 项目名称：miniAdWall（互动广告墙）
- 项目定位：一个支持视频/图片广告展示、点赞、排行的前端应用，附简单 Node/Express Mock 服务
- 角色与贡献：前端架构、组件设计、接口对接、排序策略、体验优化
- 技术栈：
  - 前端：Vite + React + TypeScript，CSS Modules
  - 状态与数据：本地组件状态 + 自定义 hooks，异步请求封装（`src/services/api.ts`）
  - 工具：ESLint、tsconfig 分层
  - 后端（本地 Mock）：Express，静态资源与 `ads.json` 数据源

## 3. 功能要点与截图脚本
- 广告列表展示：卡片式布局（组件：`components/AdCard.tsx`）
- 视频播放与弹窗：`VideoPlayerModal.tsx`；支持封面点击播放
- 详情弹窗：`AdModal.tsx`，展示标题、描述、链接、时长等
- 点赞与计数：前端本地计数（可扩展到后端 API）
- 排行策略：`utils/ranking.ts` 自定义排名（可说明算法权重/可扩展性）
- 数据拉取：`services/api.ts` 从 `server/ads.json` 读取（可强调接口抽象，易替换真实后端）

> 建议准备 2–3 张关键页面截图或录屏：
> 1) 首页广告墙；2) 播放器弹窗；3) 排行/点赞变化效果。

## 4. 架构与目录
- 前端入口：`src/main.tsx`（挂载/样式入口）
- 根组件：`src/App.tsx`（路由-less 单页、状态提升）
- 样式：`src/App.css` + 组件内样式
- 类型定义：`src/types.ts`（广告实体定义）
- 服务层：`src/services/api.ts`（fetch 封装，可替换为真实 REST）
- 工具层：`src/utils/ranking.ts`（排名算法抽象，便于单测/拓展）
- 服务器：`server/index.js` 提供静态与数据服务，资源位于 `server/uploads/`

## 5. 亮点与难点
- 排名算法抽象：可根据播放量/点赞权重调整，便于 A/B 测试
- 弹窗与播放器解耦：播放器/详情分组件，降低耦合，便于接入第三方播放器
- 数据层可插拔：服务层隔离，未来可切换 GraphQL/真实后端无需改动 UI 层
- 性能与体验：懒加载媒体、基础错误处理与加载态（如有）
- 类型安全：TypeScript 全链路类型约束，减少运行时错误
